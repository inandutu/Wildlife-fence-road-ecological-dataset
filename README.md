# Wildlife Fence Dataset Processing Tools

## Overview

This repository contains Python scripts for processing the Wildlife Fence Image Dataset. The tools support annotation conversion, mask generation and merging, and extraction of GPS metadata from image EXIF records.

The repository accompanies the Wildlife Fence Image Dataset hosted on Mendeley Data.

## Scripts

### 1. generate_masks.py

This script converts polygon annotations stored in VIA JSON format into binary masks suitable for machine learning workflows.

#### Functionality

* Reads VIA JSON annotations.
* Extracts polygon coordinates.
* Generates binary masks using OpenCV.
* Creates an image-wise directory structure.
* Stores individual object masks separately.

Generated structure:

```text
image_001/
├── images/
│   └── image_001.jpg
└── masks/
    ├── image_001_1.png
    ├── image_001_2.png
    └── image_001_3.png
```
#### Output

Each mask contains:

* 0 = Background
* 255 = Annotated object

The script generates instance-level masks and does not automatically merge masks into semantic segmentation masks.

### 2. extract_gps_metadata.py

This script extracts image metadata using ExifTool.

#### Extracted Information

* Capture image name
* GPS latitude
* GPS longitude

The extracted information is written to:

```text
gps_data.csv
```

#### Requirements

Install ExifTool before running the script.

Official Website:

https://exiftool.org

## Annotation Format

Annotations were created using the VGG Image Annotator (VIA).

Each object is represented by a polygon:

```json
{
  "all_points_x":[...],
  "all_points_y":[...]
}
```

These polygons are converted into binary masks for segmentation tasks.

## Machine Learning Applications

The dataset supports:

### Classification

* Single fence
* Double fence

### Object Detection

Polygon annotations can be converted to bounding boxes.

### Instance Segmentation

Uses individual masks generated by the script.

### Semantic Segmentation

Individual masks may be merged into a single mask during preprocessing using merge_masks.ipynb.

Suitable models include:

* U-Net
* DeepLabV3+
* SegFormer
* Mask R-CNN
* YOLO Segmentation
